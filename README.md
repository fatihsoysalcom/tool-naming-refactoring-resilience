# Tool Naming Refactoring Resilience

This example demonstrates two different naming patterns for software tools or services and their impact on refactoring resilience. It contrasts technology-specific naming, which tightly couples client code to implementation, with purpose-driven, abstract naming, which allows underlying implementations to change without affecting client code. The example simulates a multi-cloud storage scenario to highlight how abstract naming facilitates easier refactoring.

## Language

`python`

## How to Run

Save the code as `main.py`.
Run from your terminal: `python main.py`
